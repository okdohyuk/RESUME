[개발 경험에 대한 노력]

토이프로젝트 여러 개를 따로 만들기보다, 지속 통합하며 공통 소스를 사용해서 생각한 것을 빠르게 구현하기 위해 프로젝트를 시작하였습니다. 프론트와 서버는 서로 다른 언어를 사용 중입니다. Endpoint와 DTO를 두 번 작성하는 작업은 휴먼에러가 걱정되고, 시간 낭비라고 생각합니다. 동일화를 위해 [OpenAPI Generator](https://github.com/OpenAPITools/openapi-generator) 기술 채택하여 YAML로 작성한 API 명세서로 자동으로 만들도록 구성하였습니다. Github Action으로 CI/CD 자동화와 버전관리를 하고 있습니다.

[UX를 위한 노력]

모바일 친화적 UI로 어느 플랫폼에서도 사용할 수 있도록 접근성을 높이었으며, PWA로 어플리케이션의 경험을 할 수 있도록 하였습니다. 국외 접근을 유도하기 위해서 **next-i18next**를 사용해 국제화를 이룸과 동시에 JSON형식의 메시지 관리를 하고 있습니다. 자신이 사용하는 디바이스의 테마에 위화감이 생기지 않도록 미디어 쿼리로 라이트와 다크 모드 전환합니다.

[기술 문제 해결]

블로그 검색 기능 구현 중 카테고리의 다대일 관계와 태그 일대다관계에서 페치 조인과 배치 사이즈 조절로 검색과 페이징을 수행하였습니다.

Planetscale의 SQL를 사용 중에 취미티어 종료 소식에 따라, [MyDumper](https://github.com/mydumper/mydumper)로 GCP의 Cloud SQL로 마이그레이션 하였습니다.

## 자기소개

24년 7월 1일부터 근무할 회사를 찾고 있습니다.

[더 나아지기를 원하는 고민]

지식의 성장은 되도록 문제를 해결하기 위함입니다. 대회에 참가하기 위해 React와 React Native를 학습하였습니다. 실무에서 활성 사용자 지표상승을 목표로 SEO 최적화에 고민하였습니다. 정적 프론트엔드 페이지에서 서버가 필요한 환경으로 바뀌자, 무중단 배포를 위해 Jenkins, Docker, 클라우드 컴퓨팅 기술을 도입하였습니다. 개인프로젝트에 동적쿼리등 복잡한 로직의 서버를 만들기 위해 Spring과 JPA등을 학습하였습니다. 신기술이 나왔다며 무작정 도입하기보다는 그 기술을 개인적으로 사용해 보고 이해하여 정말 도움이 되는 기술인지 판단하고 있습니다.

[바꾸려하는 고민]

쇼핑몰에서 비슷한 상품을 비교하는 경우 `우클릭 후 새 창에서 링크 열기` 하거나 `Ctrl + 클릭`으로 여러 상품을 띄어서 비교하는 경우가 많습니다. onClick 이벤트나 Nextjs의 Link의 경우 클릭을 해야 함수가 동작하기 때문에 시맨틱 하지 못합니다. 사소한 동작까지도 고민하여 사용자가 다른 데는 되는데 왜 여기는 안되냐며 불편이 들지 않도록 고민합니다.

[팀의 성장도 나의 성장]

형식적인 코드리뷰보다 동료와 고민하는 페어프로그래밍을 좋아합니다. 혼자서 끙끙대며 만든 결과물을 타인이 따로 시간을 들여 설명을 해주고, 그 부분을 수정하면 흐름이 자주 끈기게 됩니다. 페어 프로그래밍은 서로 같은 내용에 대해 고민하고 빠른 피드백으로 자주 쓰는 단축키까지 물어 볼 수 있어서 실제로도 빠르게 성장하는 경험을 하여 중요하다고 생각합니다. 

기능을 구현하기 위해 리서치한 내용은 문서화하여 공유합니다. 관심 있는 프로젝트가 준비되어 있더라도 언제나 모든 프로젝트에 관여할 수는 없는 법입니다. 프로젝트를 진행하면서 겪었던 과정들과 학습한 내용을 정리해 두어서 팀원이 문서를 읽고 학습할 수 있다면, 여러모로 선한 영향력은 물론이고, 프로젝트를 교환하게 되었을 때 인수인계에 필요한 시간도 줄어듭니다. 나중에 시간이 흘러서 관련자에게 물어볼 수 없게 되더라도 왜 그런 선택을 했는지 확인하는데 도움이 될 것입니다.
